# 🛡 ربات فروش VPN تلگرام

ربات فروشگاهی حرفه‌ای تلگرام برای فروش خودکار اشتراک VPN، متصل به پنل **[GuardCore](https://github.com/erfjab)** با دیتابیس **Supabase**. این پروژه شامل دو ربات مجزاست که هم‌زمان اجرا می‌شوند:

- **ربات کاربر** 🛒 — خرید اشتراک، مشاهده اشتراک‌ها، کیف پول و افزایش موجودی
- **ربات ادمین** 🔧 — مدیریت کاربران، پلن‌ها، شارژ کیف پول و تایید رسیدهای پرداخت

---

## ✨ امکانات

### ربات کاربر
- 🛒 **خرید اشتراک** — انتخاب پلن، پیش‌فاکتور، پرداخت از کیف پول و دریافت آنی لینک اشتراک
- 📦 **اشتراک‌های من** — مشاهده وضعیت، حجم مصرفی و تاریخ انقضای هر اشتراک
- 👛 **کیف پول** — مشاهده موجودی و افزایش موجودی با کارت‌به‌کارت
- 💳 **افزایش موجودی** — انتخاب مبلغ (یا مبلغ دلخواه)، دریافت اطلاعات کارت، ارسال تصویر رسید و اطلاع‌رسانی خودکار پس از تایید ادمین
- ♻️ **بازگشت خودکار وجه** — اگر ساخت اشتراک در پنل با خطا مواجه شود، مبلغ به کیف پول برمی‌گردد

### ربات ادمین
- ➕ **افزودن کاربر** — ساخت اشتراک برای هر کاربر با آیدی تلگرام (با نام کاربری دلخواه Guard)
- 🔍 **جستجوی کاربر** — مشاهده اطلاعات، موجودی و اشتراک‌های یک کاربر
- 👥 **مدیریت کاربران** — لیست صفحه‌بندی‌شده کاربران با دکمه‌های فعال‌سازی / غیرفعال‌سازی / حذف اشتراک
- 💰 **شارژ کیف پول** — افزایش دستی موجودی کاربر با اطلاع‌رسانی خودکار به او
- 📦 **مدیریت پلن‌ها** — افزودن، ویرایش (نام، حجم، مدت، قیمت، سرویس‌ها) و حذف پلن‌ها بدون نیاز به دستکاری دیتابیس
- 🧾 **بررسی رسیدها** — دریافت خودکار رسیدهای کارت‌به‌کارت با دکمه تایید / رد (تصمیم‌گیری اتمیک؛ بدون امکان تایید دوباره)

---

## 🏗 معماری

```
کاربر تلگرام ──► ربات کاربر ──┐
                              ├──► Supabase (کاربران، پلن‌ها، تراکنش‌ها، رسیدها)
ادمین تلگرام ──► ربات ادمین ──┘
                              └──► پنل GuardCore (ساخت و مدیریت اشتراک‌ها)
```

هر دو ربات در یک پروسه‌ی `main.py` و به‌صورت هم‌زمان (asyncio) اجرا می‌شوند و از طریق ماژول `bridge` به یکدیگر پیام می‌فرستند (مثلا اطلاع‌رسانی تایید رسید از ربات ادمین به کاربر).

## 📋 پیش‌نیازها

| نیازمندی | توضیح |
|---|---|
| **Python 3.11+** | زبان اجرای پروژه |
| **دو ربات تلگرام** | از [@BotFather](https://t.me/BotFather) دو توکن بگیرید (یکی برای کاربران، یکی برای ادمین) |
| **پنل GuardCore** | یک پنل فعال Guard به‌همراه API Key |
| **حساب Supabase** | یک پروژه رایگان در [supabase.com](https://supabase.com) کافی است |

## 🚀 نصب و راه‌اندازی

### ۱) دریافت سورس و نصب وابستگی‌ها

```bash
git clone https://github.com/ParixG/Vpn-Bot.git
cd Vpn-Bot

python -m venv .venv
# لینوکس / مک:
source .venv/bin/activate
# ویندوز:
.venv\Scripts\activate

pip install -r requirements.txt
```

### ۲) ساخت دیتابیس در Supabase

وارد پروژه Supabase شوید، از منوی **SQL Editor** محتوای فایل [`schema.sql`](schema.sql) را اجرا کنید. این فایل جدول‌های زیر را می‌سازد:

- `users` — کاربران و موجودی کیف پول
- `plans` — پلن‌های فروش (نام، حجم، مدت، قیمت، سرویس‌های Guard)
- `subscriptions` — اشتراک‌های ساخته‌شده
- `transactions` — دفتر کل تراکنش‌ها (واریز / خرید)
- `topup_requests` — درخواست‌های افزایش موجودی و وضعیت تایید

> 💡 چند پلن نمونه هم در انتهای فایل درج می‌شود؛ مقدار `guard_service_ids` را با شناسه سرویس‌های واقعی پنل خودتان جایگزین کنید (بعدا از داخل ربات ادمین هم قابل ویرایش است).

### ۳) تنظیم متغیرهای محیطی

فایل `.env example` را به `.env` تغییر نام دهید و مقادیر را پر کنید:

```env
USER_BOT_TOKEN=توکن ربات کاربران
ADMIN_BOT_TOKEN=توکن ربات ادمین
ADMIN_IDS=123456,7891234

GUARD_URL=https://panel.example.com/
GUARD_API_KEY=کلید API پنل Guard

SUPABASE_URL=آدرس پروژه Supabase
SUPABASE_KEY=sb_...

CARD_NUMBER=6037-0000-0000-0000
CARD_HOLDER=نام دارنده کارت
```

| متغیر | توضیح |
|---|---|
| `USER_BOT_TOKEN` | توکن ربات فروشگاه (سمت کاربر) |
| `ADMIN_BOT_TOKEN` | توکن ربات مدیریت |
| `ADMIN_IDS` | آیدی عددی ادمین‌ها، جداشده با کاما — فقط این افراد به ربات ادمین دسترسی دارند |
| `GUARD_URL` | آدرس کامل پنل GuardCore |
| `GUARD_API_KEY` | کلید API پنل |
| `SUPABASE_URL` / `SUPABASE_KEY` | مشخصات اتصال به Supabase (از بخش Project Settings → API) |
| `CARD_NUMBER` / `CARD_HOLDER` | اطلاعات کارت بانکی که برای کارت‌به‌کارت به کاربر نمایش داده می‌شود |

> 🆔 برای پیدا کردن آیدی عددی تلگرام خود می‌توانید از رباتی مثل [@userinfobot](https://t.me/userinfobot) استفاده کنید.

### ۴) اجرا

```bash
python main.py
```

هر دو ربات با هم بالا می‌آیند. ربات ادمین را استارت کنید تا منوی مدیریت را ببینید و ربات کاربر را برای تست خرید امتحان کنید. 🎉

## 🖥 اجرای دائمی روی سرور (systemd)

برای اجرای همیشگی روی سرور لینوکسی، یک سرویس systemd بسازید:

```ini
# /etc/systemd/system/vpnbot.service
[Unit]
Description=VpnPluser Telegram Bot
After=network-online.target

[Service]
WorkingDirectory=/opt/Vpn-Bot
EnvironmentFile=/opt/Vpn-Bot/.env
ExecStart=/opt/Vpn-Bot/.venv/bin/python main.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now vpnbot
sudo journalctl -u vpnbot -f   # مشاهده لاگ‌ها
```

```
│   └── handlers/        #   خرید، اشتراک‌ها، کیف پول، افزایش موجودی
├── admin_bot/           # ربات ادمین
│   └── handlers/        #   افزودن کاربر، جستجو، مدیریت پلن، بررسی رسید و ...
├── db/                  # لایه دسترسی به Supabase
├── guard/               # کلاینت API پنل GuardCore
└── utils/               # ابزارهای مشترک (فرمت اعداد، دکوراتورها و ...)
```

## 🔒 نکات امنیتی

- فایل `.env` حاوی توکن‌ها و کلیدهاست — **هرگز آن را در گیت کامیت نکنید** (در `.gitignore` قرار دارد).
- دسترسی به ربات ادمین فقط برای آیدی‌های موجود در `ADMIN_IDS` باز است.
- عملیات کیف پول (کسر، شارژ، تایید رسید) با توابع اتمیک دیتابیس انجام می‌شود تا شرایط رقابتی (race condition) و تایید دوباره رسید ممکن نباشد.
